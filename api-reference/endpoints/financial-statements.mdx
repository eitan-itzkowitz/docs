Field Definitions
lines
Each financial statement is structured into individual lines, representing account groups, accounts, or rollup categories.

Key Properties
Field	Type	Description
id	string	Unique identifier for this statement line.
parentId	string or null	If present, this line belongs to a parent grouping. Null for top-level lines.
childIds	array of string	IDs of child lines under this line.
depth	integer	Depth level in hierarchy. Used for indentation.
name	string	Human-readable label (e.g., "Total Revenue", "Shipping Costs").
type	enum	rollup (aggregated totals), parent_account_group (subgroup of accounts), or account (individual account).
amountType	enum	credit (e.g., revenue) or debit (e.g., expenses).
intervals	array	Reported values for each requested period.
Monetary Values
Uses the MonetaryValue model:
{
  "amount": 1234.56,
  "currency": "USD"
}
Supports positive and negative values (negative = refunds, expenses).
Currency follows ISO 4217 standard (e.g., USD, EUR).
