# Retrieve Financial Statements

<TryIt />

### **GET** `/financial-statements`

[!tip] Live  
This endpoint is fully operational.

---

## **Request Parameters**

| Parameter   | Type                 | Required | Description |
|------------|----------------------|----------|-------------|
| `start_date`  | `string (YYYY-MM-DD)` | **✅ Required** | Start date of the statement period. |
| `end_date`    | `string (YYYY-MM-DD)` | **✅ Required** | End date of the statement period. |
| `type`        | `string`             | **✅ Required** | Type of statement: `balance-sheet`, `income-statement`, `cash-flow`. |

---

## **Response Schema**  

### **FinancialStatement Object**

| Field         | Type             | Required | Description |
|--------------|----------------|----------|-------------|
| `type`       | `string`         | **✅ Required** | The financial statement type (e.g., `balance-sheet`). |
| `start_date` | `string (YYYY-MM-DD)` | **✅ Required** | Start date of the period. |
| `end_date`   | `string (YYYY-MM-DD)` | **✅ Required** | End date of the period. |
| `lines`      | `array of objects` | **✅ Required** | List of statement line items. |

---

### **Line Object**

| Field       | Type               | Required | Description |
|------------|--------------------|----------|-------------|
| `id`       | `string`           | **✅ Required** | Unique identifier for the statement line. |
| `parentId` | `string or null`   | **❌ Optional** | Parent line grouping (if applicable). |
| `childIds` | `array of strings` | **❌ Optional** | Child lines under this grouping. |
| `depth`    | `integer`          | **✅ Required** | Depth level for hierarchy indentation. |
| `name`     | `string`           | **✅ Required** | Display label (e.g., `"Total Revenue"`). |
| `type`     | `enum`             | **✅ Required** | `rollup` (totals), `parent_account_group`, `account`. |
| `amountType` | `enum`            | **✅ Required** | `credit` (revenue) or `debit` (expenses). |
| `intervals` | `array`            | **✅ Required** | Reported values per period. |

---

## **Monetary Values**
Uses the **MonetaryValue** model.

### **MonetaryValue Object**
| Field      | Type    | Required | Description |
|-----------|--------|----------|-------------|
| `amount`  | `number` | **✅ Required** | Financial amount with up to 2 decimals. |
| `currency` | `string` | **✅ Required** | ISO 4217 currency code (`USD`, `EUR`). |

:::info  
- **Supports positive & negative values** (negative = refunds, expenses).  
- **Currency follows ISO 4217 standards**.  
:::
