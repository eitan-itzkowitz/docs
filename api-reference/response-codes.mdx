# API Response Codes

The Puzzle API provides structured response codes to indicate the status of requests. Below, you can toggle between versions to see the latest changes.

<Tabs>
<Tab title="v1.0">

## Error Response

If an error occurs during a request, the response follows this format:

```json
{
    "errors": [
        {
            "message": "jwt expired", // Human-readable error message
            "code": "UNAUTHENTICATED" // Standard error code
        }
    ]
}
```

### Error Reference

| HTTP Status | Error Code            | Description |
|------------|----------------------|-------------|
| 400        | `BAD_USER_INPUT`      | Invalid input provided (missing fields, invalid values, etc.). |
| 401        | `UNAUTHENTICATED`     | Missing, invalid, or expired authentication credentials. |
| 403        | `FORBIDDEN`           | Insufficient permissions to access the resource. |
| 404        | `RESOURCE_NOT_FOUND`  | No resource exists for the given ID. |
| 409        | `RESOURCE_PROCESSING` | Resource is still syncing in Puzzle; try again later. |
| 410        | `RESOURCE_GONE`       | Resource was removed or cannot be accessed. |
| 422        | `RESOURCE_LOCKED`     | Resource is locked or deactivated. |
| 500        | `INTERNAL_SERVER_ERROR` | Unexpected server error; retry or contact support. |

## Response Codes and Workflows

### Asynchronous Processing

For large or complex requests, the API returns:
1. `202 ACCEPTED` with a **status URL** for polling results.
2. Webhook notifications when processing is complete.

### Success Messages

| HTTP Status | Code      | Description |
|------------|----------|-------------|
| 200        | `OK`     | Request successful; result included. |
| 201        | `RECEIVED` | POST request acknowledged and queued. |
| 202        | `ACCEPTED` | Processing asynchronously; includes polling URL. |

### Error Messages

#### Technical Errors

| HTTP Status | Code               | Description |
|------------|-------------------|-------------|
| 500        | `SERVER_ERROR`      | Unexpected server error. |
| 502        | `GATEWAY_ERROR`     | Forwarding request failed. |
| 504        | `TIMEOUT`           | Request timed out; retry with exponential backoff. |
| 429        | `TOO_MANY_REQUESTS` | Rate limit exceeded; retry later. |

#### Authorization Errors

| HTTP Status | Code         | Description |
|------------|-------------|-------------|
| 401        | `UNAUTHORIZED` | Invalid or insufficient API key permissions. |
| 403        | `FORBIDDEN`    | Valid key, but restricted access. |

#### Pipeline and Validation Errors

| HTTP Status | Code                | Description |
|------------|--------------------|-------------|
| 400        | `MALFORMED`         | Invalid request format or missing fields. |
| 402        | `UNRECOGNIZED`      | Unknown parameters (e.g., invalid company ID). |
| 404        | `NOT_FOUND`         | Resource does not exist. |
| 422        | `BUSINESS_VALIDATION` | Request violates accounting/business rules. |

### Webhook and Polling

- **Polling URL:** Provided in `202 ACCEPTED` responses.
- **Webhook Notifications:** Configurable in Puzzle settings, retries up to 3 times.
